# Система памяти ZCode ADE — Инструкция на первые дни

## Что это такое?

Система памяти — это твоя личная база знаний, которая запоминает всё:
- Команды, которые ты используешь
- Знания, которые ты находишь
- Паттерны и рецепты
- Опыт из рабочих сессий

Всё хранится в **ChromaDB** (база данных на твоём компе, никуда не уходит) и связано **графом связей** (NetworkX).

## День 1: Наполняем память

### Способ 1: Кнопка "+ New" (самый простой)

1. Открой вкладку **Memory**
2. Нажми кнопку **"+ New"** (вверху справа)
3. Заполни:
   - **Content** — что хочешь запомнить (любой текст)
   - **Tags** — теги через запятую (например: `powershell, git, полезное`)
   - **Source** — откуда (необязательно)
4. Выбери тип записи (кнопки слева):
   - **Knowledge** — знания и факты
   - **Pattern** — паттерны и рецепты
   - **Command** — команды и рецепты CLI
   - **Project** — информация о проектах
   - **Template** — шаблоны
   - **Session** — записи сессий
5. Нажми **Submit**

Готово! Запись сохранена в ChromaDB.

### Способ 2: Загрузка документа (умная)

Если у тебя есть файл `.md` или `.txt` с заметками — скорми его системе!

```powershell
cd C:\Users\stsgr\.zcode\Zai-agent-toolkit\tools

# Предпросмотр (не сохраняет, просто показывает что извлечёт):
python doc_intelligence.py extract "C:\путь\к\файлу.md"

# Загрузить один файл (regex — быстро, но шумно):
python doc_intelligence.py ingest "C:\путь\к\файлу.md"

# Загрузить с AI (умнее, с переводами на русский, без мусора):
python doc_intelligence.py ingest "C:\путь\к\файлу.md" --llm

# Или все файлы из папки:
python doc_intelligence.py batch "C:\путь\к\папке\с\заметками"
```

Что произойдёт:
- Система **автоматически** извлечёт из документа:
  - Термины и определения
  - Инструкции и шаги
  - Команды CLI
  - API-эндпоинты
  - Теги
- Каждое извлечение станет отдельной записью в памяти
- Между записями создадутся **связи в графе**

**Regex vs LLM**:

| Режим | Скорость | Качество | Мусор |
|-------|----------|----------|-------|
| regex (по умолч.) | 1-2 сек | Поверхностно | Много лишнего |
| `--llm` | 10-30 сек | Глубоко, с переводами | Минимум |

**Рекомендация**: для важных документов используй `--llm`, для быстрого сканирования папки — regex.

**Пример**: файл с 10 командами и 5 терминами создаст ~15 записей и ~20 связей в графе.

### Что делать с HTML-файлами?

Система понимает только `.md` и `.txt`. HTML-файлы нужно сначала конвертировать:

```powershell
# Конвертировать HTML в текст (убрать все теги):
(Get-Content "C:\путь\файл.html" -Raw) -replace '<[^>]+>', '' -replace '&nbsp;',' ' -replace '&amp;','&' -replace '&lt;','<' -replace '&gt;','>' | Out-File "C:\путь\файл.txt" -Encoding utf8

# Потом загрузить:
python doc_intelligence.py ingest "C:\путь\файл.txt"
```

### Как чистить мусор из памяти

Regex-режим иногда создаёт мусорные записи (имена файлов, короткие обрывки). Чистим:

```powershell
cd C:\Users\stsgr\.zcode\Zai-agent-toolkit\tools

# Удалить конкретную запись:
python memory_cli.py delete knowledge_20260519_002642

# Массовая чистка мусора:
python -c "
from memory_cli import get_client
client = get_client()
coll = client.get_collection(name='knowledge')
result = coll.get()
junk = []
for i, doc_id in enumerate(result['ids']):
    c = result['documents'][i]
    if len(c) < 30 and c.endswith('.md'): junk.append(doc_id); continue
    if len(c) < 20 and c.startswith('STD-'): junk.append(doc_id); continue
    if c in ['string','number','boolean','function','object','array','void','any']: junk.append(doc_id); continue
    if len(c) < 15: junk.append(doc_id); continue
if junk:
    coll.delete(ids=junk)
    print(f'Удалено мусора: {len(junk)}')
    print(f'Осталось: {coll.count()}')
else:
    print('Мусора нет!')
"
```

## День 2: Граф и связи

### Просмотр графа

1. Открой вкладку **Graph** в дашборде
2. Видишь сеть точек и линий? Это твой граф знаний
3. Каждая точка — запись, каждая линия — связь

### Типы связей

| Связь | Что значит |
|-------|-----------|
| `same_session` | Записи из одной сессии |
| `defines_term` | Документ определяет термин |
| `has_instruction` | Документ содержит инструкцию |
| `has_command` | Документ содержит команду |
| `tagged_with` | Запись помечена тегом |
| `imports` | Файл импортирует другой файл |
| `depends_on` | Проект зависит от пакета |
| `parent_dir` | Файл лежит в папке |

### Команды графа

```powershell
cd C:\Users\stsgr\.zcode\Zai-agent-toolkit\tools

# Статистика графа:
python memory_cli.py graph stats

# Найти соседей записи:
python memory_cli.py graph neighbors knowledge_20260519_120000

# Найти путь между двумя записями:
python memory_cli.py graph query-path --from session_1 --to knowledge_1

# Поиск по графу:
python memory_cli.py graph search "chromadb"

# Визуализация в HTML (откроется в браузере):
python memory_cli.py graph viz --format html --open

# Экспорт графа для дашборда:
python memory_cli.py graph export --output my-graph.json
```

## Частые вопросы

**В: Память сама наполняется?**
О: Нет, автоматически — только сессии через `auto-save-session.ps1`. Остальное ты добавляешь сам: через "+ New", через загрузку документов, или через CLI. Система не лезет никуда без тебя.

**В: А зачем мне вручную добавлять?**
О: Потому что только ты знаешь, что важно. Зато когда добавишь — система запомнит навсегда и найдёт по смыслу, а не просто по ключевым словам.

**В: Что будет если запустить `doc_intelligence.py ingest` дважды на одном файле?**
О: Дубликаты не создадутся — система проверяет по похожести (dedup).

**В: Русский текст поддерживается?**
О: Да! ChromaDB и все инструменты работают с кириллицей. В браузере дашборда всё отображается нормально. В PowerShell могут быть `??????` вместо русского — это проблема консоли Windows, не системы. Исправляется: `[Console]::OutputEncoding = [System.Text.Encoding]::UTF8`

**В: Где хранятся данные?**
О: На твоём компе: `C:\Users\stsgr\.zcode\memory\chromadb\` — это база ChromaDB, и `C:\Users\stsgr\.zcode\memory\graph.json` — граф связей. Никуда не отправляются.

**В: Как удалить запись?**
О: `python memory_cli.py delete <id_записи>` — id видно в дашборде или через `list`.

**В: Что если что-то сломалось?**
О: Переинициализируй базу: `python memory_cli.py init`. Внимание: это пересоздаст коллекции, старые данные сохранятся.
