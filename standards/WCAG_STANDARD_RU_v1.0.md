# Стандарт: Доступность WCAG 2.1 AA v1.0

> ID: STD-A11Y-001
> Version: 1.0
> Level: **[C] Critical**
> Reference: https://www.w3.org/TR/WCAG21/

---

## 1. Принцип: Воспринимаемость (Perceivable)

### 1.1 Контраст текста (1.4.3 — AA)

Минимальные коэффициенты контраста:

| Тип элемента | Обычный текст (< 18px / < 14px bold) | Крупный текст (>= 18px / >= 14px bold) |
|--------------|--------------------------------------|----------------------------------------|
| Минимум (AA) | 4.5:1 | 3:1 |
| Улучшенный (AAA) | 7:1 | 4.5:1 |

**Правило:** Каждая пресет-тема ДОЛЖНА проходить AA для ВСЕХ пар токенов цвета, используемых в тексте.

**Проверяемые пары на тему:**

- foreground на background
- muted на background
- muted-foreground на card
- border-foreground на background
- destructive на background

### 1.2 Контраст нетекстовых элементов (1.4.11 — AA)

UI-компоненты и графические объекты ДОЛЖНЫ иметь контраст >= 3:1 относительно смежных цветов.

Применяется к:

- Границы и фоны кнопок (относительно окружения)
- Границы инпутов (относительно фона)
- Индикаторы фокуса (относительно фона)
- Индикаторы выбора, переключатели, чекбоксы
- Иконки состояния (не декоративные)

### 1.3 Изменение размера текста (1.4.4 — AA)

Текст ДОЛЖЕН масштабироваться до 200% без потери контента или функциональности.

- Использовать единицы `rem` для font-size, НЕ `px`
- Контейнер не должен переполняться при 200% масштабе
- Без `overflow: hidden` на контейнерах текста

### 1.4 Reflow (1.4.10 — AA)

Контент ДОЛЖЕН переформатироваться в одну колонку при ширине 320px без горизонтальной прокрутки.

- Без фиксированных ширин > 320px на блоках контента
- Использовать responsive-утилиты (max-w-*, w-full)
- Тест: ширина браузера = 320px, без горизонтальной прокрутки

### 1.5 Изображения и медиа (1.1.1 — A)

- Все `<img>` ДОЛЖНЫ иметь атрибут `alt`
- Декоративные изображения: `alt=""` + `role="presentation"`
- Информативные изображения: описательный `alt` текст
- SVG-иконки: `aria-hidden="true"` когда декоративные, `role="img"` + `aria-label` когда информативные

## 2. Принцип: Управляемость (Operable)

### 2.1 Навигация с клавиатуры (2.1.1 — A)

Все интерактивные элементы ДОЛЖНЫ управляться с клавиатуры.

| Элемент | Клавиша | Действие |
|---------|---------|----------|
| Button/Link | Enter / Space | Активация |
| Tab/Option | Tab | Перемещение фокуса |
| Tab list | Left/Right arrow | Переключение таба |
| Listbox | Up/Down arrow | Навигация по опциям |
| Dialog | Escape | Закрытие |
| Accordion | Enter / Space | Переключение |
| Menu | Up/Down arrow, Escape | Навигация, закрытие |

**Правило:** Никакого `onClick` без клавиатурного эквивалента. Если используется `onClick` на неинтерактивном элементе, добавить `role="button"` + `tabIndex={0}` + `onKeyDown`.

### 2.2 Видимость фокуса (2.4.7 — AA)

Каждый фокусируемый элемент ДОЛЖЕН иметь видимый индикатор фокуса.

```css
/* Глобальный стиль фокуса — применяется через Tailwind */
*:focus-visible {
  outline: 2px solid var(--ring);
  outline-offset: 2px;
}
```

- Индикатор фокуса ДОЛЖЕН иметь контраст >= 3:1 относительно фона
- НИКОГДА не использовать `outline: none` без альтернативного стиля фокуса
- Цвет фокус-ринга: `tokens.ring` из активного пресета темы

### 2.3 Порядок фокуса (2.4.3 — A)

Порядок фокуса ДОЛЖЕН быть логичным и предсказуемым.

- Порядок Tab следует визуальному порядку (сверху-вниз, слева-направо)
- Модальные окна: фокус-ловушка — Tab цикличен внутри модала
- После закрытия модала: фокус возвращается к триггер-элементу
- Skip navigation: первый фокус Tab идёт на skip-ссылку

### 2.4 Сенсорные цели (2.5.5 — AAA, рекомендуется AA)

Все интерактивные элементы ДОЛЖНЫ иметь минимальную сенсорную цель 44x44px.

- Кнопки, ссылки, инпуты: минимум `h-11 w-11` (44px)
- Если визуальный размер < 44px, добавить padding до 44px цели
- Кнопки только с иконкой: минимум 44x44px с `aria-label`

### 2.5 Skip Navigation (2.4.1 — A)

Каждая страница ДОЛЖНА иметь ссылку skip navigation как первый фокусируемый элемент.

```tsx
<a href="#main-content" className="sr-only focus:not-sr-only focus:absolute ...">
  Перейти к основному содержимому
</a>
```

Для layout-ов с sidebar также предоставить:

```tsx
<a href="#sidebar" className="sr-only focus:not-sr-only ...">
  Перейти к навигации
</a>
```

## 3. Принцип: Понятность (Understandable)

### 3.1 Язык (3.1.1 — A)

Страница ДОЛЖНА иметь атрибут `lang` на `<html>`.

```tsx
// layout.tsx
<html lang="ru">  // или "en" в зависимости от контента
```

### 3.2 Идентификация ошибок (3.3.1 — A)

Ошибки формы ДОЛЖНЫ быть:

1. Описаны текстом (не только красная граница)
2. Связаны с полем через `aria-describedby`
3. Анонсированы скринридерами через `aria-live` или `role="alert"`

```tsx
<div>
  <input aria-invalid="true" aria-describedby="email-error" />
  <p id="email-error" role="alert">Email обязателен</p>
</div>
```

### 3.3 Согласованная навигация (3.2.3 — A)

Навигация ДОЛЖНА появляться в том же относительном порядке на каждой странице.

## 4. Принцип: Надёжность (Robust)

### 4.1 Имя, роль, значение (4.1.2 — A)

Все UI-компоненты ДОЛЖНЫ иметь программно определяемые имя и роль.

| Компонент | Требуемые ARIA |
|-----------|----------------|
| Tab list | `role="tablist"`, tabs: `role="tab"`, panels: `role="tabpanel"` |
| Tab | `aria-selected`, `aria-controls` (tab), `aria-labelledby` (panel) |
| Accordion | `aria-expanded`, `aria-controls` |
| Dialog | `role="dialog"`, `aria-modal="true"`, `aria-labelledby` |
| Progress | `role="progressbar"`, `aria-valuenow`, `aria-valuemin`, `aria-valuemax` |
| Toggle | `aria-pressed` |
| Menu | `role="menu"`, items: `role="menuitem"` |
| Listbox | `role="listbox"`, options: `role="option"`, `aria-selected` |
| Slider | `role="slider"`, `aria-valuenow`, `aria-valuemin`, `aria-valuemax` |
| Checkbox | `aria-checked` (если кастомный) |
| Radio | `aria-checked` (если кастомный) |

### 4.2 Статусные сообщения (4.1.3 — AA)

Статусные сообщения ДОЛЖНЫ анонсироваться скринридерами без получения фокуса.

```tsx
// Toast-уведомления
<div role="status" aria-live="polite">{message}</div>

// Критические алерты
<div role="alert">{errorMessage}</div>
```

## 5. Движение и анимация

### 5.1 prefers-reduced-motion (2.3.3 — AAA, рекомендуется)

Все анимации ДОЛЖНЫ уважать `prefers-reduced-motion`.

```css
@media (prefers-reduced-motion: reduce) {
  *, *::before, *::after {
    animation-duration: 0.01ms !important;
    animation-iteration-count: 1 !important;
    transition-duration: 0.01ms !important;
    scroll-behavior: auto !important;
  }
}
```

**В компонентах:**

```tsx
// Проверка через хук
const prefersReducedMotion = usePrefersReducedMotion();

// Условная анимация
<motion.div animate={prefersReducedMotion ? {} : { opacity: 1, y: 0 }} />
```

## 6. Чек-лист имплементации

Каждый компонент ДОЛЖЕН пройти этот чек-лист перед merge:

- [ ] Весь текст соответствует 4.5:1 контрасту (3:1 для крупного текста)
- [ ] Весь нетекстовый UI соответствует 3:1 контрасту
- [ ] Focus-visible ring виден на всех интерактивных элементах
- [ ] Навигация с клавиатуры работает (Tab, Enter, Space, Arrows, Escape)
- [ ] Сенсорные цели >= 44px
- [ ] aria-label на всех кнопках только с иконкой
- [ ] Корректные ARIA-роли и состояния
- [ ] Ссылка skip navigation присутствует
- [ ] Нет анимации без защиты prefers-reduced-motion
- [ ] Reflow работает при 320px без горизонтальной прокрутки
- [ ] Ошибки формы имеют текст + aria-describedby
- [ ] Атрибут lang на <html>

## 7. Требования к контрасту тем

Каждый пресет темы ДОЛЖЕН документировать свои коэффициенты контраста:

| Пара токенов | Champagne | Cyan Night | Zinc | Champagne Light | Cyan Morning |
|--------------|-----------|------------|------|-----------------|--------------|
| foreground / background | | | | | |
| muted / background | | | | | |
| muted-foreground / card | | | | | |
| destructive / background | | | | | |
| ring / background | | | | | |

Заполненные значения ДОЛЖНЫ показывать соотношение >= 4.5:1 для обычного текста.

## 8. Тестирование

### 8.1 Ручное тестирование

- Пройти Tab через всю страницу — каждый интерактивный элемент должен быть доступен
- Использовать скринридер (VoiceOver/NVDA) — каждый элемент должен анонсироваться корректно
- Масштаб 200% — контент не должен теряться
- Установить viewport 320px — без горизонтальной прокрутки
- Включить "Reduce Motion" в настройках ОС — анимации должны остановиться
