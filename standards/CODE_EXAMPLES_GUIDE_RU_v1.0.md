## Единый образовательный документ (полная версия)

**Цель документа:** Научиться создавать примеры кода, которые:
- Понятны с первого взгляда
- Корректно копируются без лишних символов
- Запускаются без доработок
- Безопасны для случайного использования
- Доступны для всех пользователей (включая использующих экранные читалки)

---

## Часть 1. Базовые принципы оформления

### 1.1. Синтаксическая подсветка (Syntax Highlighting)

**Что это:** Разные цвета для ключевых слов, строк, комментариев.

**Как указать в Markdown:**
````markdown
```python
def greet(name):
    # Это комментарий
    return f"Hello, {name}!"
```
````

**Почему важно:** Без подсветки код сливается в серую массу, глаза устают.

**Ограничение:** Не полагайтесь ТОЛЬКО на цвет для передачи смысла — экранные читалки не видят цвета.

---

### 1.2. Форматирование кода (Code Formatting)

**Что это:** Правильные отступы, пробелы, переносы строк.

**Плохо:**
```python
def add(x,y):return x+y
```

**Хорошо:**
```python
def add(x, y):
    return x + y
```

**Инструменты:** Prettier, Black (Python), gofmt (Go), ESLint (JavaScript).

---

### 1.3. Соблюдение стиля (Code Style)

**Что это:** Единые правила для всего проекта или языка.

**Пример для Python (PEP 8):**
- 4 пробела для отступа
- Максимум 79 символов в строке
- Имена: `snake_case` для функций, `CamelCase` для классов

**Плохо:**
```python
def CalculateSum( A, B ): return A+B
```

**Хорошо:**
```python
def calculate_sum(a, b):
    return a + b
```

## Часть 2. Принципы копирования и использования

### 2.1. Готовый к копированию пример (Copy-paste ready)

**Что это:** Код можно скопировать и вставить без удаления лишних символов.

**❌ Плохо (есть лишние символы):**
```python
5: def hello():
6:     print("Hi")
```

```bash
$ pip install requests
$ python script.py
```

**✅ Хорошо (чистый код):**
```python
def hello():
    print("Hi")
```

```bash
pip install requests
python script.py
```

---

### 2.2. Самодостаточный пример (Self-contained / Isolated)

**Что это:** Код работает сразу после вставки, не требует внешних файлов, сети или базы данных.

**❌ Плохо (зависит от отсутствующей переменной):**
```python
result = process(data)  # откуда data?
```

**✅ Хорошо (самодостаточный):**
```python
data = [1, 2, 3, 4, 5]

def process(numbers):
    return [n * 2 for n in numbers]

result = process(data)
print(result)  # [2, 4, 6, 8, 10]
```

---

### 2.3. Идемпотентный пример (Idempotent)

**Что это:** Пример можно запускать много раз — результат каждый раз одинаковый.

**❌ Плохо (зависит от времени или счётчика):**
```python
import time
print(time.time())  # каждый раз разное
```

**✅ Хорошо (предсказуемый результат):**
```python
print(2 + 2)  # всегда 4
```

---

### 2.4. Исполняемый / тестируемый пример (Executable / Testable)

**Что это:** Код из документации автоматически проверяется при сборке.

**Пример doctest (Python):**
```python
def multiply(a, b):
    """
    >>> multiply(3, 4)
    12
    """
    return a * b
```

**Почему круто:** Пример никогда не устареет — тест упадёт при изменении кода.

## Часть 3. Показ изменений и выводов

### 3.1. Дифф / Патч (Diff) — показываем изменения

**Когда нужно:** Объясняем, что изменилось между двумя версиями.

**Как оформить:**
```diff
def calculate(x):
-    return x * 2  # старая формула
+    return x ** 2  # новая формула
```

Зелёное (`+`) — добавлено, красное (`-`) — удалено.

---

### 3.2. Блоки кода с выводом (REPL-стиль)

**Когда нужно:** Показать не только код, но и результат его работы.

**Пример:**
```python
>>> squares = [x**2 for x in range(5)]
>>> squares
[0, 1, 4, 9, 16]
```

**⚠️ Важно для копирования:** Пользователь должен копировать только строки **без `>>>`**! Лучше давать отдельно код и вывод:

````markdown
**Код:**
```python
squares = [x**2 for x in range(5)]
print(squares)
```

**Вывод:**
```
[0, 1, 4, 9, 16]
```
````

## Часть 4. Аннотации и пояснения

### 4.1. Комментарии и стрелки (Decorating Code)

**Когда нужно:** Пояснить конкретное место в длинном примере.

**✅ Хорошо (с комментариями):**
```python
def process(user_data):
    # Проверка на пустой словарь
    if not user_data:
        return None
    
    # Извлечение имени
    name = user_data.get("name", "Anonymous")
    return name.upper()
```

**✅ Хорошо (со стрелками для визуалов):**
```python
def process(user_data):
    # ⬇️ Проверка на пустой словарь
    if not user_data:
        return None
    
    # ⬇️ Извлечение имени
    name = user_data.get("name", "Anonymous")
    return name.upper()
```

---

### 4.2. Доступность (Accessibility)

**Проблема:** Не все пользователи видят цвета или используют экранные читалки.

**Правила:**
1. Не полагайтесь ТОЛЬКО на цвет для передачи смысла
2. Добавляйте текстовые комментарии
3. Используйте явные указатели (`# ВАЖНО:`, `⚠️`)

**❌ Плохо (только цвет):**
```python
# (красный текст) измените эту строку
api_key = "key"
```

**✅ Хорошо (текстовое пояснение):**
```python
# ИЗМЕНИТЕ ЭТУ СТРОКУ:
api_key = "your-key-here"
```

## Часть 5. Минимизация и чистота

### 5.1. Минимальный воспроизводимый пример (Minimal Reproduction)

**Принцип:** Пример должен быть **минимальным**, но **достаточным**.

**❌ Плохо (много лишнего):**
```python
import sys
import json
import datetime
from collections import defaultdict

x = 42  # эта переменная не нужна для примера

def main():
    print("Hello")
    # ... 50 строк кода ...
```

**✅ Хорошо (только суть):**
```python
numbers = [1, 2, 3]
result = sum(numbers)
print(result)  # 6
```

---

### 5.2. Регистрозависимость

**Проблема:** Многие языки (Python, Java, C, Go) чувствительны к регистру.

**❌ Плохо (ошибка в регистре):**
```python
Print("Hello")   # NameError: name 'Print' is not defined
```

**✅ Хорошо:**
```python
print("Hello")
```

## Часть 6. Многоязычность и версионирование

### 6.1. Многоязычные примеры (Polyglot)

Когда нужно показать **один и тот же алгоритм** на нескольких языках:

````markdown
```python
print("Hello, world!")
```

```javascript
console.log("Hello, world!");
```

```bash
echo "Hello, world!"
```
````

**Правило:** Указывайте язык у каждого блока, даже если он повторяется.

---

### 6.2. Версионирование (версии языка / библиотек)

Пример может работать в Python 3.11, но не работать в 3.7.

**Правило:** Указывайте минимальную версию.

````markdown
```python
# Требуется: Python 3.10+
match value:
    case 1:
        print("One")
```
````

## Часть 7. Сворачивание и безопасность

### 7.1. Сворачивание длинного кода (Collapsible / Foldable)

Когда пример длинный, но не весь нужен для понимания:

````markdown
<details>
<summary>📁 Полный код сервера (нажмите, чтобы раскрыть)</summary>

```python
from flask import Flask
app = Flask(__name__)

@app.route('/')
def hello():
    return "Hello"

if __name__ == '__main__':
    app.run()
```
</details>
````

**Принцип:** Скопировать можно, но длинный код не отвлекает.

---

### 7.2. Безопасность примеров

**Проблема:** Пользователь может скопировать и запустить опасный код.

**✅ Хорошо (предупреждение об опасности):**
```bash
# ⚠️ ВНИМАНИЕ: Это удалит временную папку!
rm -rf ./temp_folder
```

```sql
-- ⚠️ ОПАСНО в продакшене: удаляет все данные
-- DROP DATABASE production;
-- ↗️ Замените на безопасный пример:
DROP DATABASE IF EXISTS test_db;
```

**Правила:**
- Маркируйте опасные команды
- Используйте заглушки (test, example)
- Для деструктивных операций показывайте их закомментированными

## Часть 8. Продвинутые техники

### 8.1. Живой код (Live / Interactive)

**Что это:** Код, который можно запустить прямо в браузере.

**Форматы:**
- **MDX** (React в Markdown)
- **CodePen / JSFiddle** (встраивание iframe)
- **Jupyter Notebooks** (`.ipynb`)

**Где полезно:** Учебные платформы, документация UI-библиотек.

---

### 8.2. Автогенерация примеров из реального кода (Single source of truth)

В профессиональной документации код в Markdown **не пишут руками**, а **импортируют** из репозитория.

**Docusaurus (React):**
```jsx
import CodeBlock from '@theme/CodeBlock';

<CodeBlock language="jsx" title="/src/App.js">
  {require('!!raw-loader!../examples/App.js').default}
</CodeBlock>
```

**Sphinx (Python):**
```rst
.. literalinclude:: ../examples/quick_start.py
   :language: python
   :lines: 10-20
```

**Почему важно:** Пример никогда не устареет, потому что это настоящий работающий код из проекта.

---

### 8.3. Лицензия и атрибуция (юридический аспект)

Когда вы берёте чужой код для образовательного документа:

**Правила:**
1. Указывайте **источник** (автор, лицензия)
2. Некоторые лицензии **запрещают** или ограничивают копирование без копирайта

**Пример оформления:**
```markdown
> Пример из документации [Requests](https://docs.python-requests.org/), лицензия Apache 2.0

```python
import requests
response = requests.get('https://api.github.com')
```
```

## Практическое задание

Отформатируйте следующий пример правильно:

**Исходный (плохой):**
```
$ python
>>> def foo(x,y):
...     return x+y
... 
>>> foo(5,3)
8
```

**Требования:**
1. Убрать приглашения оболочки (`$`, `>>>`, `...`)
2. Добавить синтаксический язык
3. Сделать пример самодостаточным
4. Добавить комментарии
5. Показать код и вывод отдельно

**Ожидаемый ответ:**

````markdown
```python
def add(x, y):
    """Возвращает сумму двух чисел."""
    return x + y

# Пример использования
result = add(5, 3)
print(result)  # 8
```

**Вывод:**
```
8
```
````

## Итог: чек-лист хорошего примера кода

- [ ] Есть синтаксическая подсветка (указан язык)
- [ ] Соблюдены отступы и переносы
- [ ] Нет лишних символов (`$`, `>>>`, номеров строк, стрелок внутри)
- [ ] Пример самодостаточный (self-contained)
- [ ] Нет лишних импортов и переменных
- [ ] Есть понятные комментарии (не только цвет)
- [ ] Показан ожидаемый результат (отдельно от кода)
- [ ] Указана версия языка/библиотеки (если важно)
- [ ] Опасные команды помечены предупреждением
- [ ] (Опционально) Пример автоматически тестируется
- [ ] (Если чужой код) Указана лицензия и источник

## Глоссарий (быстрый поиск терминов)

| Термин | Что означает |
|--------|--------------|
| **Copy-paste ready** | Код копируется без удаления лишних символов |
| **Self-contained** | Не требует внешних файлов или сети |
| **Idempotent** | Повторный запуск даёт тот же результат |
| **Isolated** | Не зависит от глобального состояния |
| **Minimal reproduction** | Самый короткий пример, который показывает проблему |
| **Polyglot** | Один алгоритм на нескольких языках |
| **REPL** | Цикл "чтение-вычисление-вывод" (с `>>>`) |
| **Diff** | Показ изменений (`+`/`-`) |
| **Single source of truth** | Код генерируется из реального файла |
