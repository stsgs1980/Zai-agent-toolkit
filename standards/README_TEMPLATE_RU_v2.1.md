## Обязательные разделы

Каждый README.md должен содержать следующие разделы в указанном порядке:

| # | Раздел | Обязателен | Описание |

| 1 | Название + Описание | Да | Имя проекта и краткое описание |
| 2 | Бейджи | Опционально | Технологические бейджи (только SVG) |
| 3 | Функции | Да | Ключевые возможности |
| 4 | Технологический стек | Да | Используемые технологии |

| 8 | API Reference | Опционально | Эндпоинты, методы |
| 9 | Скрипты | Опционально | NPM/Bun команды |
| 10 | Правила разработки | Опционально | Стиль кода, ограничения технологий |

## Пример соответствия

### Правильно

```text
## Функции
- Быстрая сборка - использует Turbopack
- Типобезопасность — полный TypeScript
```

Примечание: дефис `-` используется как разделитель в технических описаниях. Тире `—` разрешено в обычном тексте внутри элементов списка согласно MARKDOWN_STANDARD, но запрещено в заголовках и блоках кода.

### Неправильно

```text
## Функции
- Быстрая сборка -> использует Turbopack
## Функции — Основные
```

Примечание: `->` как разделитель в элементах списка нарушает правило маркеров списка. Тире `—` в заголовке запрещено.
