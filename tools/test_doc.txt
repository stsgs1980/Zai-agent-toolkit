Git Workflow Cheat Sheet

Git is a distributed version control system for tracking changes in source code.
Every developer should know the core commands and workflows.

Key Terms:

Branch - A separate line of development. You can create branches to work on
features without affecting the main codebase.

Merge - Combining two branches together. Git automatically merges changes
when possible, or flags conflicts for manual resolution.

Rebase - Moving the base of a branch to a new commit. This creates a linear
history but should be used carefully on shared branches.

Stash - Temporarily storing uncommitted changes so you can switch branches
without losing work.

Fork - A personal copy of someone else's repository on GitHub. You can make
changes without affecting the original.

Pull Request - A proposal to merge your changes into another repository.
Used for code review and collaboration.

Remote - A connection to a remote repository (like GitHub). You can have
multiple remotes for different upstream sources.

How to create a feature branch:

1. Make sure you are on the main branch
2. Pull latest changes from remote
3. Create a new branch with a descriptive name
4. Make your changes and commit
5. Push the branch to remote
6. Open a pull request

Commands:

git checkout main
git pull origin main
git checkout -b feature/my-new-feature
git add .
git commit -m "Add new feature"
git push origin feature/my-new-feature

Other useful commands:

git stash save "work in progress"
git stash pop
git rebase main
git log --oneline --graph
git diff --staged

Best Practices:
- Commit often with clear messages
- Pull before push to avoid conflicts
- Use branches for all new features
- Review code before merging
- Keep main branch deployable at all times
